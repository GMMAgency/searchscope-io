01 The idea

A precise instrument, not a template.

Every brand in the portfolio shares one visual language. The chrome, the surfaces, buttons, cards, headers, footers, is strictly monochrome: ink on paper in light mode, paper on warm near-black in dark mode. Colour is reserved for one job only: carrying meaning in data. The result reads as an instrument someone trusts, and it makes every brand instantly recognisable as part of the same family without a shared logo ever appearing.

P1 · Colour

Monochrome chrome

Black, white and grey everywhere the interface or layout lives. No brand colour in buttons, links, headers or backgrounds. The accent is ink itself, and calls to action invert: black on light, white on dark.

P2 · Meaning

Colour only where it means something

The only colour anywhere is in data: green rising, red falling, amber weak, blue brand series, grey flat. If a hue appears, it is a signal, never decoration.

P3 · Type

The mono-spine

Geist Sans for big titles and hero numbers. Geist Mono for the label layer: eyebrows, buttons, table headers and every tabular number. Inter for anything read in sentences. One rule of thumb: mono for labels and numbers, Inter for sentences.

P4 · Surfaces

Hairlines over shadows

1px hairline borders separate surfaces; elevation is reserved for true overlays. Warm near-black, never pure #000 for large surfaces; a faint grain keeps them alive.

P5 · Character

Instrument motifs, used sparingly

Viewfinder corner-ticks frame one focal element per surface. A faint coordinate grid sits behind heroes and empty states. Repeat them everywhere and they stop meaning anything.

P6 · Density

Density with air

These are working documents and data tools, so they are information-dense, but every surface keeps breathing room, hairline dividers and a clear hierarchy. Dense is not cramped.

P7 · Modes

Light and dark are equal citizens

Every asset ships in both modes. Inverted cards flip, data hues switch to their legible variants. Neither mode is an afterthought; test both before anything ships.

P8 · Restraint

One focal point per surface

One hero, one framed element, one big number, one idea. Whether it is a page, a PDF spread or a LinkedIn card, restraint is what makes the system read as considered.

“If it looks like a generic dashboard, it is wrong. If it looks like a precise instrument someone trusts, it is right.”

02 One system, many brands

The system is fixed. A brand fills five slots.

A new brand does not get a new design language. It gets a wordmark, a mark, and a token sheet. Everything else, colour behaviour, typography, spacing, motifs, chart rules, document grids and voice, is shared and non-negotiable. That is what makes the portfolio feel like one studio.

03 Logo system

One grid, one weight, one wordmark rule.

Every mark in the family is drawn on the same 28-unit grid with the same stroke, so marks from different brands sit next to each other like glyphs from one typeface. The reference mark is the Searchscope viewfinder: four corner brackets finding a bright signal.

Construction rules (all brands)

  • 28-unit grid. Draw every mark in a 28×28 viewBox; straight lines only, no curves, no rings.
  • One stroke. 1.7 units at 28 (about 6% of the grid), round caps and joins; the same weight as the icon set.
  • One bright node. Each mark carries exactly one solid signal element, and may add one secondary point at 45% opacity.
  • currentColor. Marks are single-colour and tint via CSS, so one file serves both modes.
  • Legible at 16px. If the mark muddies at favicon size, simplify it.

Wordmark rule

The logotype is always a compact, one-word, lowercase setting in Geist Mono 500 with letter-spacing -0.04em, whatever the brand is called in prose. Searchscope becomes searchscope; Growth Minded Marketing becomes growthminded. Choose the compact form once and never vary it. The screen-reader name stays the full prose name.

  • No gradients, shadows or outlines on the mark; no stretching or rotating.
  • Never recolour a mark into a data hue; it is ink or paper only.
  • No rings or circles; the family is rectilinear.
  • Never set the wordmark in another face, another case, or as two words.
  • Keep clearspace of at least one mark-height around every lockup.

  • Bright hues are for marks only: dots, bars, lines, area fills. They fail contrast as small text on white (validated: 1.7 to 2.7:1), so small coloured text always uses the deep text variants (validated: 3.8 to 5.2:1 on white).
  • Never colour alone. Every coloured state ships with an arrow, a label or a value. Green and red sit close for red-green colour-blind readers, so the arrow is not optional.
  • This is a status palette, not a categorical one. For multi-series categorical charts use greys plus one hue, never a rainbow (§08).
  • Colour by meaning, not direction. A falling position number is an improvement, so it reads green.
  • Grey means flat by design; it is the one hue allowed to read as grey.

Per-brand accent (optional, discouraged)

The default stance for every brand is no accent: ink is the accent. A brand may opt into a single functional accent only with a deliberate decision, and it must ship as a pair: a bright value for dark mode and a deeper, text-legible value for light mode, plus an on-accent ink. It may colour primary actions and links only. It never appears in charts (the data palette owns chart colour), never in backgrounds, and never competes with the data hues for meaning.

Motion

160ms micro · 280ms standard · 600ms reveal, all on cubic-bezier(.22, 1, .36, 1). Entrance only: a number ticks once, a row reveals once. No perpetual animation, and everything respects reduced-motion.

  • Hairlines over shadows. One 1px border on a warm surface reads more premium than any drop shadow. Elevation is reserved for true overlays: menus, modals, toasts.
  • A faint grain (2 to 5% opacity) keeps large surfaces from feeling dead. You should barely notice it.
  • Reading widths near 1200px maximum; data tables may run full width.

07 Instrument motifs

Two motifs. One appearance each.

The corner-ticks echo the viewfinder mark; the coordinate grid evokes the field the data lives in. They are the family's signature texture, and they only work because they are rare: one framed element per surface, one grid field per page. Both are decorative, carry no meaning, and are hidden from assistive tech.

Corner-ticks · frame one focal element

The signature frame

Ink at ~34%, 1.5px, 16px arms.

Coordinate grid · behind heroes & empty states

The field

Ink at ~5%, 30px cells, masked so it fades at the edges.

  • Never put ticks on every card; the frame marks the one thing that matters on the surface.
  • The grid must read as texture, never as a table grid; always mask its edges.
  • Both motifs are aria-hidden and never carry information.

Rules

  • One y-axis, always. Two measures of different scale get two charts or an indexed base, never a second axis.
  • Thin marks. 2px lines, bars with a 2px gap between neighbours, markers at least 8px with a surface ring where they overlap.
  • Legend for two or more series, plus direct labels at line ends. One series needs no legend; the title names it.
  • Categorical series are greys plus one hue. The data palette is a status palette; it is never cycled as a rainbow. A sixth colour does not exist.
  • Text wears text tokens. Values, labels and legends stay in ink, dim or faint; a coloured mark beside them carries identity.
  • Chart surface sits a hair off the page with a 1px hairline frame; gridlines are faint and recessive; axis labels are mono, 11px, ink-faint.
  • Interactive charts get a hover layer: a crosshair with a tooltip on time series, a per-mark tooltip on bars and dots.
  • Motion is entrance-only; numbers count once. Reduced-motion turns it all off.

Rules

  • The shell is permanent. Sidebar, top bar and page header hold still; the work area is the only scroll region. Nothing marketing-shaped lives inside it: no hero, no CTA band, no footer.
  • One primary action per view, sitting in the page header. Everything else is ghost or text; destructive actions are rose text and border, solid rose only to confirm in a modal.
  • Mono is the chrome. Nav items, tabs, chips, table headers, paths and every number are Geist Mono; findings, descriptions and empty states are Inter. Titles are Geist at 18 to 22px, never display size.
  • Keyboard is a first-class citizen. ⌘K opens the command palette, every control shows the focus ring, and tables sort, select and expand without a pointer.
  • Skeletons, not spinners. Surface-2 blocks where content will land; live jobs get a mono counter and may pulse slowly. Toasts sit top right with a severity edge and a thin timer bar.
  • Detail in a drawer, decisions in a modal. Row detail slides in from the right; only confirm and create interrupt in the centre. Overlays are the only elevated surfaces (§06).
  • The colour rules do not relax. Severity and deltas follow §04, charts follow §08; the chrome stays ink even when the screen is dense with signal.
  • The login screen and empty states are where the motifs live (§07): one grid field, one framed element, nowhere else.

11 Decks & spreadsheets

Slides are posters. Sheets are ledgers.

PowerPoint and Excel are where the system runs on other people's machines, and the law does not relax there. A workbook is §08 with the chrome stripped back: mono headers, tabular figures, colour only in cells that literally report status. Where the fonts cannot travel, the deck ships as PDF and the workbook declares its fallbacks once.

Rules

  • One idea per slide. If a slide needs three points, it is three slides. The deck gets longer before a slide gets fuller.
  • The deck is not the document. Slides carry claims and exhibits; the detail lives in the A4 report (§10) that travels with the deck.
  • No theme decoration. SmartArt, WordArt, clip art, stock photos, shadows, gradient fills and built-in templates are all off. Type, hairlines and one framed title do the work.
  • A workbook is an instrument, not a canvas. Colour enters only through conditional formatting on data cells, carrying the meanings of §04; the chrome stays ink and hairlines.
  • Real numbers on every slide that makes a claim: 5,240 clicks, ↗ 12.4%. A slide with no number earns its place some other way, or it is cut.
  • One mode per deck: dark cover, light working slides is the default; an all-dark deck is fine for a stage. Never mixed slide by slide.

Do

  • Land one number per slide, set huge, with the mono delta beside it.
  • Freeze the header row, switch gridlines off, let hairlines rule the sheet.
  • Ship the deck as PDF; keep the .pptx and .xlsx as the masters.

Not this

  • SmartArt diagrams, theme gradients, drop shadows, slide transitions.
  • Rainbow tab strips, zebra striping, whole rows filled green and red.
  • Bullet walls read aloud; ten ideas on one slide.

Anatomy: what every video ships with

  1. A title card. Dark, one grid field, one corner-ticked title, a mono eyebrow naming the series (PRODUCT DEMO, LAUNCH). Two seconds, one reveal, then cut. It doubles as the poster frame.
  2. Mono context labels. Lower-thirds and chapter markers in Geist Mono, uppercase, bottom-left: INDEXATION · ACME.COM. They say what is on screen, not how to feel about it.
  3. Real numbers. Every claim lands as a specific mono figure on screen (412 URLs, −34% in 28d). A beat with no number earns its place some other way, or it is cut.
  4. Captions. Inter, sentence case, burned into social cuts, sidecar (.vtt) on the master. Most viewers are muted; the video must work silent.
  5. An end card. Lockup, one CTA line, the domain. Holds at least 3 seconds. The title and end cards are the only frames that wear the motifs.
  6. One mode, held. Pick light or dark for the whole video and stay there; the product UI never flips modes mid-film.
  7. A 16:9 master plus recomposed cuts. Social crops are reframed shot by shot, never letterboxed, never stretched.

Rendered charts in motion

  • The anatomy is §08, unchanged. Monochrome chart chrome, the data palette carrying the meaning, the mono number nearby. Motion adds no new colours and no new decoration.
  • Entrance only, once. Axes and gridlines fade in first at their faint opacity; a line draws left to right over 600ms on cubic-bezier(.22, 1, .36, 1); bars grow once; the hero number ticks up once. Then everything holds still.
  • No loops, no pulsing, no shimmering gradients, no camera drift or slow zoom across data. A finished chart is a held shot.
  • One chart per shot. Two series compared is one chart (§08); two charts is two shots.
  • The mono number lands with the chart and stays put; the caption or voiceover says what the shape means: what, why, move.
  • Illustrative data must still be plausible. No fake hockey sticks; nothing the product could not genuinely show.

Motion grammar

  • Hard cuts. No dissolves, wipes, whip pans, speed ramps or parallax. Pacing comes from held shots, not transitions.
  • The UI motion tokens, scaled for film. Micro moves 160ms, standard 280ms, reveals 600ms, all on the shared ease. Nothing on screen animates longer than 600ms, and nothing animates forever.
  • Type appears in one move: a 280ms fade or small rise. No typewriter effects, no letter-by-letter builds.
  • In demos the cursor moves like a practitioner: deliberate, no shake, no giant cursor, no cartoon click effects. What is clicked gets a beat before and after.
  • Sound is understated: a quiet bed, no build-drops, no whooshes on cuts. The voiceover is the practitioner voice of §14, British English, real numbers, no hype.

Do

  • Draw the chart once, land the mono number, hold the shot.
  • Cut on the action: the click, then the result, at real speed.
  • Name the beat in a mono lower-third and let the UI speak.

Not this

  • Perpetual background motion: particles, gradients, floating cards.
  • Slow zooms and parallax drifts across data.
  • Rainbow dashboards, typewriter text, build-drop music.

14 Voice

Practitioner, plain-spoken, specific.

The copy sounds like a senior practitioner explaining what they found, not a brand performing enthusiasm. British English throughout. Real numbers beat adjectives: 412 URLs, −34% in 28d, pos 6.

  • No em dashes, ever. Use commas, colons, semicolons or parentheses. This applies to copy, documents and code comments alike.
  • No hype words, no exclamation marks, no emoji in product or brand copy.
  • Findings follow one shape: what, why, move. “Crawled, not indexed. Thin templates diluting the cluster. Consolidate or prune.”
  • Controls say exactly what happens: “Create brief”, then “Brief created”. Errors say what went wrong and how to fix it.
  • Speak to the reader's world, not the system's internals: people manage pages and queries, not database rows.

Write this

  • Your CTR fell 0.4pt while impressions rose. The title is the suspect: test a rewrite.
  • 214 indexation findings, 12 critical. Start with the 3 blocking your money pages.
  • Search never stops moving, so neither does the read.

Not this

  • Unlock game-changing insights with our revolutionary AI-powered platform!
  • Oops! Something went wrong.
  • Supercharge your SEO journey today.

15 Adopting a new brand

From name to shipped identity in eight steps.

  1. Fix the names. The prose name, and the compact one-word lowercase logotype in Geist Mono 500. Decide once; never vary.
  2. Draw the mark on the 28-unit grid: straight lines, 1.7 stroke, one bright node, optional faint point, currentColor. Check it at 16px.
  3. Pick the accent stance. Default is ink. Opting into a brand accent requires the bright/deep pair and the restrictions in §04.
  4. Copy the token sheet below and set only the brand slots. Neutrals, data palette and type stay untouched.
  5. Wire both modes and test them side by side. The inverted patterns must flip cleanly.
  6. Produce the asset kit: lockup, mark and app icon, each in ink and on-dark, SVG and PNG, wordmark outlined to paths.
  7. Build the first three templates: the OG / link-share image, the LinkedIn 4:5 card, and the A4 report shell. Everything else derives from these; a brand with a product also stands up the app shell (§09).
  8. Audit against §01. One focal point per surface, motifs used once, colour only in data, no em dashes. If it looks like a template, revise until it looks like an instrument.
